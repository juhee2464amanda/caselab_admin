## 1. 컨셉 (한 줄 정의)

**케이스랩 (Caselab)** — 부제: *일잘러의 AI 실험실*

> **"AI, 다들 쓰는데 나만 못 쓰는 것 같다면 — 내 일에 어떻게 쓸지, 실험하며 찾아가는 공간"**

**핵심 철학**: "AI, 어렵게 생각하지 말고. 우리가 일하는 framework 안에서 AI 역할을 정의해 활용하자."

- AI는 도구. 어떻게 쓸지는 **일을 푸는 구조(framework)** 를 알아야 정해진다.
- 단순 프롬프트 모음 ❌ — 항상 **framework × AI 실행** 짝으로 다룬다.
- 정체성: **실험하며 찾아가는** (≠ 같이 만들기). `같이`는 글 보이스로만.
- 자체 용어: **"일을 푸는 방법"**.

**다른 채널과의 차별점**:

| 채널 | 포지션 |
|---|---|
| 칼릭스 | 프롬프트 디테일 — AI가 잘 답하게 만드는 법 |
| 하프던 | AI 트렌드 빠른 소개 |
| 마케터임지은 | 마케팅 직무 특화 |
| **케이스랩** | 일의 framework 먼저 → 단계별 AI 실행 → 직접 실험한 솔직한 결과 |

## 2. 누구를 위한 서비스인가

페르소나 5명 (유저 인터뷰 1568건 인용 기반). **공통 페인**:
- 자기의심 — "다들 쓰는데 나만 못 쓰는 것 같다"
- 결과물이 "그럴듯하지만 쓸 수 없다" (톤·맥락 안 맞음)
- AI가 오히려 시간을 잡아먹는다
- AI 강의·콘텐츠 자체에 대한 깊은 불신

| 페르소나 | 한 줄 |
|---|---|
| A 박지현 (마케팅 기획자) | 빈 입력칸 앞에서 머리가 하얘짐 |
| B 이민준 (전략팀) | AI가 검증이 아니라 정당화만 함 |
| C 김소연 (1인 창업/프리랜서) | 강의에 데이고도 FOMO로 또 결제 |
| D 최현수 (영업팀장) | 결과물 톤이 윗사람한테 안 맞음 |
| E 정다은 (스타트업 마케터) | 단순 업무는 OK, 복잡 업무에서 막힘 |

## 3. 콘텐츠 — 두 트랙

| 트랙 | 다루는 것 | 상세 구조 | 발행 |
|---|---|---|---|
| **AI 트렌드** | 최신 기능·트렌드·기능 분해 | 3단: 뭐가 나왔나 → 직접 실험 → 언제 쓸만·별로 | 주간 |
| **실전 케이스** | 기획서·고객 인터뷰 정리 등 실제 업무 케이스 | 4단: 본질 → 일 푸는 순서(Framework) → 단계×AI 실행 → 솔직 후기 | 격주~월간 |

**콘텐츠 spec**: "별로였던 것" ≥30% 분량 / 시간 라벨(읽는데·적용) 필수 / 초기 콘텐츠는 한 직무군 집중.

**콘텐츠 제작 방식**: 실전 케이스는 운영자가 직접 실험·작성 (솔직 후기가 정체성의 핵심 — AI 대필 ❌). AI 트렌드는 운영자가 AI로 초안 작성 → **검수·팩트체크** → 등록 (콘텐츠 부담 완화). MVP는 외부 AI로 초안 후 붙여넣기, 빌트인 "AI 초안" 버튼은 Phase 1.5.

## 4. 핵심 기능 (MVP)

- **메인 홈** — 이번 주 큐레이션 + 최신 콘텐츠 (매거진식)
- **두 트랙 목록** + **콘텐츠 상세** (4단/3단, 복사 프롬프트 블록)
- **저장 / 👍👎 / 의견 보내기**
- **"이런 거 어때요"** — 운영자 후보 주제 투표 + 의견 폼
- **무료 전자책** — 전자책 상세 → 로그인 → 주문서(이름·연락처·이메일) → 이메일로 받기 (리드 마그넷)
- **운영자 admin** — 콘텐츠 CRUD·입력 폼·분석(GA4 병행)·가입자·전자책 관리·의견함·후보 카드 관리

## 8. 채널 & 수익화

- **채널**: 인스타그램 메인 → 웹 허브. 페르소나 B·D는 인스타 밖이라 브런치·뉴스레터 보조 채널 필요.
- **수익화**: 초기 무료. **무료 전자책**(리드 마그넷)은 MVP 포함 — 로그인 → 주문서(이름·연락처·이메일) → 이메일 발송. 주문 흐름 자체를 MVP에서 가동. **유료 전자책은 Phase 2** — 같은 주문 흐름에 PG 결제 단계만 추가.

## 9. 운영자 포지션 & 운영 원칙

**운영자 포지션**
- 커리어는 전면에 내세우지 않는다 (13년 외국계·대기업 실무 — 영업·마케팅·고객조사·전략·데이터 분석). 깊이는 콘텐츠에서 자연스럽게 묻어나게.
- 톤: 전문가가 아닌 **"나도 배우는 중인데, 직접 써봤더니 이렇더라"** — 보이스의 1인칭 따뜻함과 일치.

**운영 원칙**
- 첫 콘텐츠가 서비스의 첫인상 — 제일 자신 있는 업무 케이스로 시작.
- 발행 속도보다 퀄리티 — 트렌드 빠른 채널이 아니라 검증된 케이스 채널.
- 인스타와 웹의 톤 일관성 — 같은 사람이 운영한다는 느낌.
- 출시 시 콘텐츠 10개 이상 쌓고 오픈 (메인이 비어 보이지 않게).
