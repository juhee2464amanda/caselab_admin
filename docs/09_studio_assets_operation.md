# 콘텐츠 스튜디오 — 프롬프트·가이드 등록 운영 초안

2026-07-07 · 스튜디오 소메뉴(바로쓰는 프롬프트 / 공식 가이드) 신설에 맞춘 운영 제안.

## 1. 왜 스튜디오 소속인가

본가의 `/prompts`(바로 쓰는 프롬프트)와 `/guides`(공식 가이드)는 케이스·트렌드 같은 "읽는 콘텐츠"가 아니라
**복사·클릭 한 번으로 소비되는 자산(asset)** 이다. 둘 다 `tools` 테이블 하나에서 나간다:

| 본가 화면 | 쿼리 | 본문 계약(body jsonb) |
|---|---|---|
| /prompts | `tools` · `category='prompt'` · published | `prompt`(복사 본문), `promptCategory`(think/make/verify/refine), `source`, `sourceUrl` + `pick_order` 컬럼 |
| /guides | `tools` · `category in ('guide','context-card')` · published | `guideCategory`(prompt/cases/education/skills/agents), `source`, `sourceType`(default/github/course) + `url`·`description` 컬럼 |

씨앗→기획→발행이 롱폼(케이스·트렌드)의 라인이라면, 프롬프트·가이드는 **짧은 큐레이션 라인**이다.
같은 "만들어서 내보내는 곳"이므로 콘텐츠 스튜디오 아래 탭으로 묶는다.

- 작업실: 씨앗 → 롱폼 초안 → 발행 → 홈배치 (기존)
- 바로쓰는 프롬프트: 자산 등록 데스크 (신규 `/admin/prompts`)
- 공식 가이드: 링크 큐레이션 데스크 (`/admin/guides` 이동 배치)

## 2. 등록 화면 설계 (이번 PR 반영분)

**공통 원칙: 폼 필드 = 본가 노출 계약. 운영자가 jsonb를 몰라도 되게.**
(감사에서 지적된 tool body 계약 문제의 해소 — 폼이 계약을 강제한다)

### 바로쓰는 프롬프트 (`/admin/prompts`)
- 구 리다이렉트(`/admin/contents?type=prompt`)를 전용 데스크로 교체.
- 필드: 제목* / 분류*(사고·만들기·검증·다듬기) / 프롬프트 본문*(복사 대상) / 출처 라벨 / 출처 URL / PICK 순서.
- 목록은 본가와 같은 감각: **에디터 PICK 밴드**(pick_order 오름차순) → 분류별 그룹.
- 추가 즉시 published (1인 운영, 본가 revalidate 60s).

### 공식 가이드 (`/admin/guides`)
- 분류 입력을 자유 텍스트(`job_tags[0]`)에서 **본가 5탭 select(`body.guideCategory`)** 로 교체.
  - 기존 등록분은 `guideCategory` 미지정 → 본가에서 전부 '프롬프트 작성법' 탭으로 노출되는 상태.
    **백필 필요: admin에서 각 항목 [수정]→분류 지정→저장 한 번씩** (몇 건 안 되면 이걸로 충분).
- 출처 라벨(`body.source` + `job_tags[0]` 호환 유지), 자료 유형(`body.sourceType`: 공식 문서/GitHub/강의) 추가.
- 썸네일 커스텀(`thumbLabel/thumbBg/…`)은 body 병합으로 보존 — 필요해지면 폼 확장.

## 3. 운영 리듬 제안

### 소싱 → 등록 루프
1. **HERMES 씨앗에서 승격**: scout 브리핑에 담긴 프롬프트 팁·공식 문서 링크는 롱폼 씨앗이 아니라
   자산 후보다. triage 때 "이건 프롬프트감/가이드감"이면 스튜디오 탭에서 바로 등록하고 씨앗은 반려 처리.
   (후속: 씨앗 카드에 "프롬프트로 등록" 원클릭 — 이미 seed→tool 생성 파이프가 있어 확장 용이)
2. **롱폼에서 추출**: 케이스 글에 쓴 프롬프트가 반응 좋으면 완결형으로 다듬어 프롬프트 자산으로 승격.
   글→자산 재활용이 가장 싼 콘텐츠 소스.
3. **공식 문서 정기 순회**: OpenAI/Anthropic/Google 문서·cookbook 업데이트를 월 1회 훑고 가이드 갱신.

### 큐레이션 기준
- 프롬프트: "복사 한 번이면 끝"이 서비스 약속 — **붙여넣고 빈칸만 채우면 돌아가는 완결형만**.
  변수는 `[대상]`, `[목적]` 식 대괄호 표기 통일. 직접 1회 이상 실행해 본 것만 등록(=검증된).
- 가이드: 공식(벤더 직접 작성) 또는 업계 표준 GitHub(스타 수 기준)만. 블로그 아님.
- 4분류/5탭 균형: 특정 탭이 3개 미만이면 그 탭 우선 소싱.

### 노출 운영
- PICK은 3~5개 유지, 격주 교체. 새로 미는 프롬프트를 PICK 1로, 밀려난 건 pick_order 비워 일반으로.
- 분기 1회 가이드 링크 점검(404·이동 확인) — 링크 자산은 썩는다.
- 페이스 제안: 프롬프트 주 2~3건, 가이드 주 1~2건. 뉴스레터 "이번 주 프롬프트" 코너와 연동하면
  등록 리듬이 발행 리듬을 만든다.
